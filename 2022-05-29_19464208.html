기획 회의 (13:00 ~ 19:00)

실시간 코드리뷰 vs 강의

  • 강의는 code cast로도 충분할 것 같다.

    • 우리가 여기서 더 나은 퀄리티가 있는 서비스를 만들 수 없을 것 같다.

  • 코드리뷰 쪽으로 다시 생각해보자.

각자 의견 정리

주형

Code Cast좋다. 하지만 이 기능 하나로 가기엔 애매하다.

  • 기획 통과가 불가능할 것

차라리 코드 리뷰로 돌아가거나, 페어프로그래밍으로 집중하는게 좋을 듯.

  • 드라이버, 네비게이터

  • 애자일은 빼고

코드 리뷰는 좀 진부하다 생각한다

  • 코드리뷰 기능 핵심은 code cast기능일 것 가은데 이건 좀..

페어프로그래밍 다시 디벨롭 하는게 좋을 것 같다.

  • 여기에 code cast도 잘 해보면 추가 할 수 있지 않을까..?

  • 포커스를 기획 심의에 맞추자

  • 타겟층이 애매하지만 할 말은 있을 것 같다.

페어 프로그래밍.. → 하령님의 말이 맞다고 생각. 의견수정

  • MVP 생각해서 코드리뷰로 가자!

하령

  • 우선 code cast 는 힘들 것 같다. 메인으로 두기에 좀 애매한 것 같다.

  • 페어프로그래밍, 코드리뷰 둘 다 괜찮다.

    • 둘 중에 실 사용자를 확보할 수 있는게 좋을 것 같다. (기획 심의 통과 < MVP 만들어서 사용자 피드백)

    • 그걸 생각해보면 코드리뷰가 좋을 것 같다.
