팀장의 고민사항

배경

  • 왜 안 읽씹을 할까?

    • 읽게 되면 내가 계속 답장을 해야 될까봐 즉 해당 메세지의 중요성을 파악하지 못한다

  • 왜 사람들은 안 읽씹을 할지 고민해보면 좋을 것 같음

  • 톡 서비스를 만들어야 한다

모각코 플랫폼

배경

  • 모각코 할 때 특정 플랫폼을 사용하지 않음.

  • 디코, 줌 많이 쓴다

    • 공부랑 맞지 않음

해결

  • 코딩 중점

  • 주요 타겟: 개발자

    • 스크린쉐어가 효과적

      • 화면 공유 필요

    • 시간 단위 잔디 그래프

  • 백준이 재밌는 이유

    • 업적/뱃지 시스템

      • 공부를 많이하면 발급

      • 여러명에서 하면 인싸 업적 달성

    • 랭킹(티어) 시스템

      • 눈으로 성장을 확인

  • 음악봇

  • 채널

  • 메타버스

    • 캐릭터가 공부

  • 개발자를 위한 서비스

    • 코덕(유행 지남)

    • wakatime(코드짠 것 통계)

노트 공유

배경

  • 공식 문서나 표준을 이해하기 힘들다

  • 다른 사람의 노트도 공유하면서 볼 수 있으면 좋을 것 같았음

  • 블로그에 정리된 글을 한곳에서 볼 수 있으면 좋을 것 같았음

해결

  • 특정 주제에 대해 노트/주석을 달 수 있다

    • 공식 문서(NextJS, RFC문서, …) 같은 것을 그대로 클론할 수도 있다

    • 사람들을 초대해서 개인 노트 공간을 만들 수도 있다

  • 이상한 주석은 비추 기능으로 필터링

Geo메모

어떤 장소에 가서 직접 메모를 남긴다

배경

  • 서서 메뉴판을 봐야할까?

  • 정보가 너무 많다

    • 장소에 갔을 때 정보를 실시간으로 얻을 수 있다면 좋겠다

  • 사람들은 후기와 메모를 엄청 남기지만 다 기억하기 어렵다

해결

  • 사람들은 걸어다니며 현재 위치에 메모

  • 추천/비추천 기능

  • 가게에 들리면 메뉴판이 뜬다

줄서기 싫어

맛집은 가고 싶지만 줄은 서기 싫다!

배경

캐치 테이블(예약하는 것)과 다르게 직접 가서 먹고 싶은 경우

  • 줄이 엄청 긴 경우가 있다

  • 재고를 다 써서 영업 종료한 경우가 있다

    • 특히 라멘집

  • 영업 시간이 네이버에 적힌 시간이랑 다르다

가게의 입장에서도 사람이 너무 많아 폭주하면 힘들다.

해결

  • 대기줄을 알려준다

  • 재고가 다 했으면 끝났음을 알려준다

  • 통계를 내서 언제 폭주하는지 알 수 있다

단점

  • 다른 연수생님의 아이디어와 겹칠 수도 있다

    • 카페 자리 남는지 알려주는 서비스
