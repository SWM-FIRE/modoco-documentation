 KPT 회고

Keep - Keep Doing

Problem - Stop doing

Try - Start doing

  • 실행력 칭찬해

    • 일정은 맞췄다

  • 이벤트 개최할 때 컨셉 부족했다

    • 목적 불분명

  • 기획서 유지보수 하자

    • 방법: 문서

  • 출근 잘했다
    (지각 미리 말해줌)

  • 기획에서 지라 사용 애매…

  • 이벤트 개최할 때 기획 먼저 확실히 하고 목적을 분명하게 하자 (최소 2주전)

  • confluence 정리 잘했다

  • 기획서 유지보수가 안됐다

  • 다음에도 이벤트 개최는 하자

  • 의사소통 잘됐다

    • 애매한거 잘 말한다

    • sync 잘 맞춰지고 있다

  • 기획서 취합할 때 문서가 흩어져 있어서 어떤 걸 봐야할지 잘 몰랐었다

  • 한글 사자

  • 이벤트 개최하는 것 자체는 정말 좋았다

  • 기획서 작성할 때 어떤 어려움들을 겪었고 어떻게 해결했는지 까먹었다. (기억이 잘 안난다)

  • 기획할 때 지라에서는 스토리포인트나 역할분배에 덜 신경쓰자. 좀 더 포괄적으로 작성해도 괜찮을 것 같다. (어디서 무엇을 작업하고 있는 지는 서술해주자)

  • Jira 하위이슈 본문 내용을 잘 안적었다.

  • 앞으로 개발할 때 마주치는 모든 이슈들, 기술적인 어려움 다 작성하자 + 해결방안

  • 보고서에 날짜만 있는게 많다

    • 컨텐츠 내용이 무엇인지 미리 확인하기가 어렵다

  • Jira 템플릿을 수정하자

    • 할것과 결과물을 필수적으로 작성

    • 문제점(오류, 버그 등) 및 해결방안은 자유롭게

  • 문서 제목에 날짜만 기입하지 말자

    • 날짜 + 간단하게 키워드라도 적어주자

지라 레이블도 적어주기

채워야겠다는 아무런 효능이 없다.

중요 일정에 대한 알람 봇을 만들어서 항상 고려하게끔 해야 한다.

회의록을 의미있는 단위에 링크를 넣어야 한다. 날짜만 적기 보다 날짜 뒤에 최소한 필요한 것 정도는 적어야 한다.

회의록 쓰는 방법에 대해 문서화를 하자.
