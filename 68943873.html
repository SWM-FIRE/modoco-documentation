TEAM FIRE (윤영기 고주형 이하령)

 스프린트 어땠나요

  • 발표 마지막 날이여서 운이 좋았다.

  • 그래서 좀 여유로웠다.

  • 만족

  • 조금의 아쉬움

    • 기획 설명

    • 공감할수 있게, 수치로

    • 시장 규모

  • 발표 후

    • ‘모여서 각자’ 를 이해 못했다.

    • 모각코 시장 규모 조사가 필요했다.

    • 앞으로 모든것을 수치적으로 증명해야 한다.

    • 심사위원이 인정할 수 있을만큼

    • 근데 이게 현실적으로 가능하나..? 막막하다.

    • 라이브 쉐어 1억명이 쓴다는 것으로 사람들이 자신의 코드를 공유한다는 것을 확실히 알려주면 좋을 듯

    • 알맹이가 없는 느낌.

    • 컨셉이 없다.

    • 한 방에 최대한 들어올 수 있는 인원을 4명으로 하고 컨셉을 잡자.

    • 교육으로 갈지 문화로 갈지 생각을 해봐야 될것 같다. (역할기능, 선생님 얘기를 심사위원이 너무 좋아했음)

    • 미디어 서버를 붙여야 할까?

 KPT 회고

Keep - Keep Doing

Problem - Stop doing

Try - Start doing

  • 출석율 좋았다

  • 기획서 유지보수가 필요하다

  • 기획 이야기를 주기적으로 하자

  • 역할 분배 잘했다

  • 심의 피드백 전혀 반영하지 않음

  • 앞으로 추가할 기능들이 진짜 필요한지 생각해보자 (ex. 라이브 캔버스, 코드 공유)

  • 기능 가지치기

  • 진짜 필요한 것 생각하고

  • 발표 전략 잘짰다

  • 디자이너와 이야기해보자.

  • 발표 잘했다

  • Q&A 잘 대처한듯..?

 Action items

  • 데모 끝날 때마다 스크럼 마스터가 기획에 대한 안건을 제시하자. (본인 의견과 안건제시, ex. 요건 이렇게 하면 어때요?)
  • expert와 디자이너 관련하여 이야기해보자. (비용, as)
  • 릴리즈 준비하자
