이번주 스프린트 어땠나요

  • 방 입장 후 화면공유가 좀 어려웠음

  • 공부가 필요

Keep

  • 시간을 잘 맞췄다

  • 목표한 바를 다 채웠다

  • 밸런스가 괜찮았다

    • 힘들었던 날이 없었다

  • 주 1회 재택이 좋다

  • 저번주보다 api 소통이 원활했다

Problem

  • 재택할 때 디코에서 같이 하고 싶다!

    • 단절되어 있는 느낌이 든다

  • 이슈 댓글을 제대로 안다는 것 같다

    • 상황 공유를 이슈 기반 댓글로 하지 않은 것 같다

  • 분량 조절이 잘 안된거 같다

    • 양이 너무 적었다

  • AWS 소통이 안됐다

  • 같은 task에 대해 완료를 여러번 했다

    • 완료를 했는데 다시 진행중으로 넘어간 경우가 많았다

  • 하위 이슈 범위가 너무 작은 것 같다

    • 하위 이슈를 몰아서 완료시킨것 같다

  • 작업 공유를 하지 않고 진도를 나갔다

Try

  • 재택할 땐 디코(추후 모도코)에서 하자

  • 의사표현을 확실히 하자

  • 백엔드와 합의할 게 있으면 같이 소통을 하자

    • 모르는게 있어서 의사 표현이 힘들어도 말하자

  • 이슈를 왜 마무리 했는지 달았으면 좋겠다

  • 하위 이슈 범위를 늘리자

  • 회의록 정리를 돌아가면서 하자

  • 진도 나가기 전에 미리 말을 하자

  • 앞으로의 스프린트는 월요일~목요일로 하자

    • 금요일은 회고

    • 토~일요일은 개인 공부 + 혹은 팀원들에게 이야기 후 프로젝트 추가 진행

Q

영기님) 먼저 끝난 사람이 다음 진도를 미리 나가도 되지 않을까?

프로젝트를 스프린트 기반으로 가다 보니까, 다음 스프린트 내용을 미리 해도 되는지, 아니면 진도에 맞춰서 리팩토링을 하는게 더 나은건지 고민된다.

하령) 이번 스프린트 안에서 맡은 역할에 충실하는게 좋지 않을까?

프론트가 두명이라 한 사람의 작업 속도가 비교적 빠를 때 애매한 것 같다. 팀원으로서 역할을 제대로 해내고 싶은데 그럴 기회가 있으면 좋겠다.
